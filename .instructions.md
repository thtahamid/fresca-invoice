# FrescaInvoice Soft-Pop Design System

## Overview
This project uses a "Soft-Pop" design aesthetic characterized by vibrant pastel colors, deep layered shadows, and large border radii. All design tokens are centralized in `src/app/globals.css` using CSS custom properties.

---

## Core Principles

### 1. **Use Existing Color Variables Only**
- **Never** hardcode colors or create new color values
- **Always** use the defined CSS custom properties from `globals.css`
- Colors are defined in OKLCH color space for perceptual uniformity

#### Available Color Variables:

**Base Colors:**
- `bg-background` / `text-foreground` - Primary background and text
- `bg-card` / `text-card-foreground` - Card surfaces
- `bg-primary` / `text-primary-foreground` - Brand/action color (blue)
- `bg-secondary` / `text-secondary-foreground` - Supporting elements
- `bg-muted` / `text-muted-foreground` - De-emphasized content
- `bg-accent` / `text-accent-foreground` - Highlighted areas
- `bg-destructive` / `text-destructive-foreground` - Errors/warnings
- `border-border` - Default borders
- `border-input` - Form field borders
- `ring-ring` - Focus rings

**Semantic Accent Colors (for visual categorization):**
- `bg-mint` - Soft green/teal - Use for product/item information
- `bg-sky` - Light blue - Use for invoice/summary sections
- `bg-lavender` - Soft purple - Use for customer/personal data
- `bg-peach` - Warm peachy-orange - Use for company/business information

**Opacity Variants:**
- Use Tailwind opacity modifiers: `bg-mint/40`, `bg-sky/30`, `bg-lavender/30`, `bg-peach/30`
- Common pattern: 30-50% opacity for card headers

---

### 2. **Shadow System (Deep Shadows)**
- **Always** use `shadow-pop` or `shadow-pop-hover` - no custom shadows
- **Never** use Tailwind's default shadow utilities (`shadow`, `shadow-lg`, etc.)

#### Shadow Usage Rules:

**`shadow-pop`** - Standard elevation:
- Use on: Cards, form fields, buttons at rest, containers
- Creates layered depth with dual-layer shadows
- Example: `className="shadow-pop rounded-xl"`

**`shadow-pop-hover`** - Elevated state:
- Use on: Hover states for interactive elements
- Larger, single-layer shadow for dramatic lift
- Example: `className="shadow-pop hover:shadow-pop-hover"`

**Common Pattern:**
```tsx
<div className="shadow-pop hover:shadow-pop-hover transition-all duration-400">
```

---

### 3. **Border Radius System (Large Radii)**
- **Match existing component patterns** for consistency
- Use the radius scale defined in `globals.css`: `--radius-sm` through `--radius-4xl`

#### Border Radius Guidelines:

**Component Type Patterns:**
- **Cards (product/invoice)**: `rounded-xl` (16px) or `rounded-2xl` (20px)
- **Form Cards**: `rounded-2xl` (20px)
- **Input Fields**: `rounded-xl` (16px) or `rounded-2xl` (20px)
- **Buttons**: `rounded-md` (10px) or `rounded-2xl` (20px) based on prominence
- **Dialogs/Modals**: `rounded-3xl` (24px)
- **Large Containers/Previews**: `rounded-[3rem]` (48px) or `rounded-3xl`
- **Badges/Pills**: `rounded-full`
- **Step Indicators**: `rounded-xl` or `rounded-2xl`

**Responsive Pattern (common):**
```tsx
className="rounded-xl md:rounded-2xl"
```

**Rule of Thumb:**
- Larger components = larger radius
- Interactive elements benefit from mid-range radii (xl, 2xl)
- Maintain visual hierarchy through radius size

---

### 4. **Hover Effects & Interactions**
- **Optional** - Use when it improves UX and provides meaningful feedback
- Not required on every interactive element

#### Recommended Hover Patterns:

**Cards (clickable/interactive):**
```tsx
className="shadow-pop hover:shadow-pop-hover hover:-translate-y-2 transition-all duration-500"
```

**Buttons (primary actions):**
```tsx
className="shadow-pop hover:scale-105 transition-all duration-300"
```

**Subtle Hover (links/secondary):**
```tsx
className="hover:bg-muted/50 transition-colors"
```

**When to Add Hover Effects:**
- ✅ Primary action buttons (generate, submit, add)
- ✅ Product cards that open dialogs
- ✅ Clickable cards that navigate
- ✅ Step indicators in multi-step flows
- ❌ Static information cards
- ❌ Disabled elements
- ❌ Text links within paragraphs (use subtle underline instead)

## Component-Specific Patterns

### Cards with Colored Headers
Pattern for sectioned cards with semantic color headers:

```tsx
<Card className="shadow-pop rounded-2xl">
  <CardHeader className="bg-peach/30 rounded-t-2xl">
    <CardTitle>Company Information</CardTitle>
  </CardHeader>
  <CardContent className="p-6">
    {/* Content */}
  </CardContent>
</Card>
```

**Color Assignment:**
- `bg-peach/30` - Company/business data
- `bg-lavender/30` - Customer/client data
- `bg-sky/30` - Invoice/summary information
- `bg-mint/40` - Product/item details

### Form Inputs
```tsx
<input 
  className="rounded-2xl border-input shadow-pop focus:ring-primary"
  type="text"
/>
```

### Buttons
```tsx
{/* Primary Action */}
<Button className="bg-primary text-primary-foreground shadow-pop hover:scale-105">
  Generate Preview
</Button>

{/* Secondary Action */}
<Button variant="outline" className="rounded-2xl shadow-pop">
  Cancel
</Button>
```

### Product Grid
```tsx
<div className="grid gap-6 md:grid-cols-2 lg:grid-cols-3">
  <Card className="shadow-pop hover:shadow-pop-hover group hover:-translate-y-2 transition-all duration-500 rounded-xl md:rounded-2xl">
    {/* Product content */}
  </Card>
</div>
```

## Transitions & Animations

### Standard Transitions
Use the custom transition utilities defined in `globals.css`:

**`transition-smooth`** - General purpose:
```tsx
className="transition-smooth" // all 0.4s cubic-bezier(0.4, 0, 0.2, 1)
```

**`transition-bounce`** - Playful interactions:
```tsx
className="transition-bounce" // all 0.5s cubic-bezier(0.34, 1.56, 0.64, 1)
```

**Tailwind Alternatives:**
```tsx
className="transition-all duration-400"  // For custom durations
className="transition-transform"         // For specific properties
```

## Responsive Behavior

### Shadows
Shadows **automatically scale** on larger screens via `globals.css` media queries:
- Mobile: Smaller shadows for subtle depth
- Tablet/Desktop (768px+): Larger shadows for dramatic pop effect

### Border Radius
Use responsive classes to increase radius on larger screens:
```tsx
className="rounded-xl md:rounded-2xl lg:rounded-3xl"
```

### Typography
Use the custom responsive utilities:
```tsx
className="text-responsive-h1"  // text-4xl → text-6xl
className="text-responsive-h2"  // text-2xl → text-3xl
```

## Special Utilities

### Glass Panel Effect
For overlays and layered UI:
```tsx
className="glass-panel"
```
- Applies backdrop blur and semi-transparent background
- Use sparingly for modals or floating elements

## Anti-Patterns (What NOT to Do)

❌ **Don't** create custom color values:
```tsx
{/* BAD */}
<div className="bg-[#FFB6C1]">
```

❌ **Don't** use default Tailwind shadows:
```tsx
{/* BAD */}
<div className="shadow-lg shadow-md">
```

❌ **Don't** use small border radii on major components:
```tsx
{/* BAD - too small for Soft-Pop aesthetic */}
<Card className="rounded-sm">
```

❌ **Don't** mix color systems:
```tsx
{/* BAD - inconsistent */}
<div className="bg-blue-500 text-mint">
```

❌ **Don't** add hover effects to everything:
```tsx
{/* BAD - overkill */}
<p className="hover:scale-105 hover:shadow-pop-hover">Static text</p>
```

✅ **Instead, do this:**
```tsx
{/* GOOD */}
<Card className="shadow-pop rounded-2xl bg-card">
  <CardHeader className="bg-sky/30 rounded-t-2xl">
    <CardTitle className="text-card-foreground">Invoice Summary</CardTitle>
  </CardHeader>
</Card>
```

## Implementation Checklist

When creating a new component, ensure:

- [ ] Colors use CSS custom properties only (no hardcoded values)
- [ ] Shadows use `shadow-pop` or `shadow-pop-hover` only
- [ ] Border radius matches similar existing components
- [ ] Semantic accent colors (mint/sky/lavender/peach) are used consistently
- [ ] Hover effects are purposeful and enhance UX
- [ ] Responsive scaling is considered (radius, typography)
- [ ] Transitions use smooth or bounce utilities
- [ ] Component follows existing patterns (cards, forms, buttons)

## Summary

The Soft-Pop design system creates a **vibrant, friendly, and highly accessible** interface through:
- **Perceptually uniform OKLCH colors** with semantic accent colors
- **Dual-layer deep shadows** that create depth and interactivity
- **Large border radii** for a modern, approachable feel
- **Purposeful hover effects** that provide meaningful feedback
- **Responsive scaling** that maintains consistency across devices

Always prioritize **consistency and visual hierarchy** when applying these patterns.
